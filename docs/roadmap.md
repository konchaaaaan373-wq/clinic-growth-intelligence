# Neco Clinic Report — ロードマップ

このドキュメントは **見直し前提の計画** です。段階・優先度・提供時期は変わり得ます。
一方、どの段階でも変わらない原則は [docs/product-philosophy.md](./product-philosophy.md) に定義しています（計画がこの原則と矛盾する場合は、原則が優先されます）。

## フェーズ1: Clinic Report Free（提供中）

- URLベースの外部準備度診断。摩擦の可視化と、MMMに必要な不足データの棚卸し。

## フェーズ2: CSVベースの初診数MMM（Clinic Report Analytics の第一歩・有料）

- **日別初診数のCSVアップロード**を起点にする（最小の実データ連携）。
- あわせて集める説明変数: 診療日/休診日、Google広告費・クリック/表示、医療コラム公開日、
  YouTube/Instagram/TikTok 投稿日・成果、ポスティング実施日/部数/費用、天気/祝日/季節性。
- これらを時系列で突き合わせ、**施策別の初診寄与とおおよそのCPAを「推定」**する。

## フェーズ3: API連携による自動化

- GA4 / Search Console / Google広告 / YouTube Analytics / Googleビジネスプロフィール などを連携し、
  データ収集を自動化する。
- 施策別の推定寄与・推定CPA・来月の施策提案・予算配分提案へ。

## Analytics プラットフォームの拡張余地

Clinic Report MMM は Analytics 上の **最初の** 分析機能であり、プラットフォームとしては
ダッシュボード・ベンチマーク・予測・AIインサイト・定期レポートなどのモジュール追加を想定しています。
いずれのモジュールも「アウトカムは日別初診数」「出力は推定であって断定ではない」という原則（P6/P10）に従います。

## レポート保存・共有の将来計画

- 現状はクライアント状態・localStorage・PDF保存（JSONは開発用）にとどめる。
- 将来的に `/reports/:id` による共有URL＋サーバー側保存、専用PDF生成を検討する。
